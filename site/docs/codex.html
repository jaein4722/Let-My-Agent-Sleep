Codex

Codex support uses same-session resume.

LMAS keeps the long-running job under tmux, lets Codex stop polling, and resumes the same task when the work finishes.

Install

npx let-my-agent-sleep install --agent codex

Restart Codex after installing the LMAS skill. The Codex session must be resumable from the machine where the job is running.

Live server and SSH mode

For an already-running TUI to wake immediately, start Codex's built-in app server and attach the TUI to it:

codex app-server --listen unix://
# From the terminal where you use Codex:
codex --remote unix://

Keep the first command running while you use the attached TUI. This works locally and over SSH. LMAS connects to Codex's existing server; it does not install or start a service of its own.

What the agent should do

  • Use the installed let-my-agent-sleep skill for commands that would require waiting.
  • Start the command with LMAS and return only the handoff summary.
  • After LMAS_HANDOFF v1, end the turn instead of sleeping, polling, or repeatedly continuing.

If Codex is not available

The job continues under tmux. LMAS attempts a recovery resume and keeps a recovery prompt with the run results. If a TUI or Desktop task stayed open but did not update, reopen it before continuing.
