Claude Code

Claude Code support is experimental.

LMAS keeps the real job under tmux and registers a lightweight native background waiter. While the Claude session and waiter stay alive, completion wakes that same session without polling. If the native path is definitely unavailable, LMAS retains the existing resume fallback.

Install

npx let-my-agent-sleep install --agent claude

This installs ~/.claude/commands/let-my-agent-sleep.md and supporting scripts under ~/.claude/lmas/let-my-agent-sleep/. It does not install a global Claude skill, so OpenCode does not pick up the Claude integration through ~/.claude/skills.

How the handoff works

  1. The command starts through LMAS and returns LMAS_HANDOFF v1 with a run id.
  2. Claude starts lmas.sh await <run_id> using its native background-task option, then ends the turn immediately.
  3. The real command continues under tmux. The waiter only receives its completion event.
  4. When the waiter exits, Claude reads the task notification's output file and continues from LMAS_COMPLETION_EVENT v1.

Recovery behavior

  • If Claude exits, the real tmux job keeps running. An orphaned waiter is not allowed to claim native delivery.
  • If LMAS can prove the waiter never registered, the waiter is dead, or its exact Claude owner is dead, it uses claude --resume.
  • An atomic delivery claim prevents native and fallback completion payloads from both winning.
  • If delivery became ambiguous after a native claim, LMAS suppresses fallback to avoid duplicates and retains resume_prompt.txt.
  • Use LMAS_CLAUDE_CONTINUE=1 only when continuing the most recent Claude session in the working directory is acceptable.

Guarantee boundary

Claude Code remains experimental. Native wake requires the exact Claude process and its registered waiter to stay alive. The job itself does not share that lifetime: it remains owned by LMAS and tmux. LMAS installs no service, executable wrapper, or PATH change.
